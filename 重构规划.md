# stockWebChat 重构规划

## Context

stockWebChat 当前是一个轻量单 Agent 聊天系统（5 个工具，单 DeepSeek 模型），与 daily_stock_analysis（DSA）对比后暴露出几个核心差距：
- **每次对话都跑全流程** — 即使只是闲聊也触发 ReAct 循环，开销大
- **无分析维度控制** — 没有问财那样的快捷指令来告诉 AI 当前想分析什么
- **策略系统浅** — 只有 4 个 preset 做 prompt 注入，DSA 有 15+ 策略 YAML + 分析工具
- **数据面窄** — 缺少大盘指数和板块排名等市场全局数据

本次重构聚焦 3 个模块。不引入多 Agent 流水线（DeepSeek V4 长上下文下单 Agent 即可胜任）。

---

## 1. 快捷指令 + 策略选择 + 轻量路由

### UI 布局

输入框上方两行芯片：

```
┌──────────────────────────────────────────────────────┐
│ [📊 行情] [💰 资金] [📰 公告] [📈 技术] [🏭 板块]   │  ← 分析维度（快捷指令）
│ [📋 完整分析]                                         │
│                                                        │
│ [多头趋势] [放量突破] [缠论] [波浪理论] [均线金叉] ... │  ← 分析方法（策略）
│                                                        │
│ ┌─────────────────────────────────────────────┐       │
│ │ 输入消息...                            [发送]│       │
│ └─────────────────────────────────────────────┘       │
└──────────────────────────────────────────────────────┘
```

**两个概念正交：**
- **维度芯片**（AnalysisChips）— 告诉 LLM **分析什么维度**，注入系统 prompt
- **策略芯片**（StrategyEditor，已有但增强）— 告诉 LLM **用什么方法论分析**，可触发特定分析工具

两者可以同时选，互不冲突。

### 维度芯片设计

| 芯片 | context 值 | 注入效果 |
|------|-----------|---------|
| `📊 行情` | `quote` | 优先查实时行情 + K线 + 标的信息 |
| `💰 资金` | `capital` | 分析主力资金流向（预留，工具就绪后生效）|
| `📰 公告` | `news` | 优先搜索新闻/公告/舆情 |
| `📈 技术` | `technical` | 综合技术指标分析 + K线形态 |
| `🏭 板块` | `sector` | 分析所属板块表现 + 排名 |
| `📋 完整` | `full` | 全维度无侧重分析 |

### 策略系统增强

**现状：** `strategies/presets.json` → 4 个策略，仅做 prompt 注入

**目标（参考 DSA 模式）：**

1. **策略定义文件化** — 每个策略独立 YAML 文件（`strategies/*.yaml`），包含：
   - `name` / `display_name` / `category`（趋势/形态/框架/反转）
   - `instructions` — LLM prompt 注入内容
   - `required_tools` — 该策略需要的工具列表（可选，为空时用全量）
   - `analysis_hooks` — 分析逻辑钩子（可选，运行时注入特定分析代码）

2. **策略数量扩展** — 从 4 个扩到 12+ 个：

   | 策略 ID | 显示名 | 类别 | 说明 |
   |---------|-------|------|------|
   | `bull_trend` | 多头趋势 | 趋势 | 已有 |
   | `shrink_pullback` | 缩量回踩 | 形态 | 已有 |
   | `volume_breakout` | 放量突破 | 形态 | 已有 |
   | `risk_stop` | 风险止损 | 风险 | 已有 |
   | `ma_golden_cross` | 均线金叉 | 形态 | 新增 |
   | `chan_theory` | 缠论 | 框架 | 新增 |
   | `wave_theory` | 波浪理论 | 框架 | 新增 |
   | `event_driven` | 事件驱动 | 框架 | 新增 |
   | `hot_theme` | 热点题材 | 框架 | 新增 |
   | `growth_quality` | 成长质量 | 框架 | 新增 |
   | `expectation_repricing` | 预期重估 | 框架 | 新增 |
   | `box_oscillation` | 箱体震荡 | 形态 | 新增 |

3. **策略分析工具**（策略不只是改 prompt，还可以跑特定分析代码）

   新增 `server/agent/strategy_tools.py`，每个策略可注册专属分析函数。例如：

   | 策略 | 分析函数 | 功能 |
   |------|---------|------|
   | 放量突破 | `check_volume_breakout(symbol)` | 计算量比、突破均线检测 |
   | 均线金叉 | `check_ma_cross(symbol, short_ma, long_ma)` | 均线交叉检测 |
   | 缩量回踩 | `check_shrink_pullback(symbol)` | 缩量 + 回踩支撑位检测 |
   | 缠论 | `analyze_chan(symbol)` | 笔/段/中枢识别（实验性）|

### 轻量路由逻辑

```
IF analysis_dimensions 非空 OR strategy_ids 非空:
    注入维度 prompt + 策略 prompt
    注册策略特有工具（如果有）
    走完整 ReAct 循环
ELSE:
    第一轮 LLM 调用：只给 2 个工具（get_realtime_quote + get_klines）
        IF LLM 返回 tool_calls:
            走完整 ReAct 循环（全工具可用）
        ELSE:
            直接返回 LLM 文本回复（纯聊天，零 Agent 开销）
```

## 2. 市场数据工具

### 新增工具

在 `server/agent/tools.py` 中新增 2 个工具：

**`get_market_indices`（大盘指数）**
- 参数：`{ exchange?: "A" | "HK" | "US" }`
- 返回：`[{symbol, name, price, change_pct}]`（上证、深证、创业板、沪深300、恒指、标普500 等）
- 数据源：tickflow，指数代码映射表硬编码

**`get_sector_rankings`（行业板块排名）**
- 参数：`{ top_n?: number, order?: "asc" | "desc" }`
- 返回：`[{sector_name, change_pct}]`（默认涨幅前 10）
- 数据源：tickflow 优先，兜底 AkShare

### 修改文件

| 文件 | 改动 |
|------|------|
| `server/agent/tools.py` | 新增 2 个工具定义 + handlers |
| `server/agent/runner.py` | `_build_tool_summary` 增加新工具摘要 |

## 验证方式

1. **快捷指令**: 点芯片 → 发送 → 日志确认维度 prompt 注入正确
2. **策略分析**: 选"放量突破"→ 发送 → 确认调用了 volume_breakout 分析工具
3. **轻量路由**: 无芯片无策略的普通聊天 → 确认不走全流程
4. **市场工具**: `get_market_indices` 返回指数数据；`get_sector_rankings` 返回板块排名
5. **构建**: `cd client && pnpm build` 通过；`uv run python main.py` 启动正常
